# 📝 Notes App (MVC)

Невеликий застосунок для створення та керування короткими нотатками з використанням MVC-архітектури та localStorage.

---

## 📌 Ідея

Створити застосунок, який дозволяє:

- ➕ Додати нотатку  
- ⭐ Відмітити як “важливо”  
- 🗑 Видалити  
- 🧹 Очистити всі  
- 💾 Зберігати дані в localStorage  

---

# 1️⃣ Функціонал (MVP)

## Створення нотатки

Поля:

- `title` (обов’язково, мінімум 3 символи)
- `category` (одна з: `work`, `study`, `personal`)

Після submit нотатка додається в список.

---

## Перемикач “Важливо”

- Кнопка **Mark important / Unmark important** на кожній нотатці.
- Важлива нотатка візуально виділяється (CSS-класом).

---

## Видалення однієї нотатки

- Кнопка **Delete** на кожній карточці.

---

## Очистити всі

- Кнопка **Clear all**.
- Якщо список порожній — показати повідомлення:

```
No notes yet
```

---

## Персистентність

Після перезавантаження сторінки нотатки мають залишатись (через localStorage).

---

# 2️⃣ Обов’язкова структура (MVC)

## 📂 Структура файлів

```
index.html
style.css
js/app.js
js/notes/Model.js
js/notes/View.js
js/notes/Controller.js
```

---

## 🧠 Model

Відповідає за:

- Масив нотаток
- Роботу з localStorage
- CRUD-операції:

```
create(data)
readAll()
toggleImportant(id)
delete(id)
clearAll()
```

---

## 🎨 View

Відповідає за:

- Рендер списку
- Рендер однієї карточки
- Очищення контейнера
- Показ повідомлення **No notes yet**

---

## 🎮 Controller

Відповідає за:

- Підписку на submit/click
- Виклики Model
- Оновлення View
- Ініціалізацію при `DOMContentLoaded`

---

# 3️⃣ Модель даних

```js
{
  id: Number,
  title: String,
  category: "work" | "study" | "personal",
  important: Boolean,
  createdAt: String // ISO date
}
```

---

# 4️⃣ Мінімальна HTML-розмітка

## Форма:

- input title
- select category
- кнопка **Add note**

## Контейнер списку:

```html
<div id="notesList"></div>
```

## Кнопка:

- **Clear all**

## Блок/текст:

- **No notes yet** (показується, коли список порожній)

---

# 5️⃣ Валідація (обов’язково)

- `title.trim().length >= 3`
- `category` тільки з дозволених значень

При невалідних даних:

- Нотатка не створюється
- Вивести просту помилку під формою або `alert`

---

# 6️⃣ Технічні вимоги

- Використовувати Bootstrap
- ES Modules (`import/export`)
- Event delegation для кнопок у списку
- Код має бути розбитий по класах: `Model / View / Controller`
- Без дублювання DOM-логіки в Controller (рендер тільки у View)

---

# 7️⃣ Acceptance Criteria

- ✅ Можна додати нотатку
- ✅ Можна видалити конкретну нотатку
- ✅ Можна переключити important
- ✅ Працює Clear all
- ✅ Дані зберігаються після reload
- ✅ Є повідомлення **No notes yet** при порожньому списку
- ✅ Є базова валідація
- ✅ Проєкт має MVC-структуру

---

# 8️⃣ Бонус (+level)

- 🔎 Фільтр: All / Important / Category
- 🔃 Сортування: новіші зверху/знизу
- 🔢 Лічильник: `Total: X | Important: Y`
- ✏️ Кнопка Edit title (редагування через prompt або modal)
