1. Об’єкт user
   Створіть об’єкт user з такими полями:

name — ім’я<br>
age — вік (number)<br>
email<br>
isSubscribed — boolean<br>
balance — string, наприклад "150.25"<br>
verified — string "1" або "0"

Завдання:
Створіть змінну з числовою версією balance.<br>
Створіть змінну з boolean-версією verified.<br>
На основі даних user визначте, чи має користувач доступ до сервісу. <br>
Доступ є, якщо: <br>
 - вік 18+,<br>
 - акаунт підтверджений,<br>
 - є підписка або позитивний баланс.<br><br>
Порівняйте age і такий самий вік у вигляді рядка через == і через ===.<br>
Збережіть результати у дві різні змінні.<br>
Виведіть результат доступу + результати порівнянь.<br>
Якщо користувачу менше 18 — збережіть у змінну текст "Access restricted due to age".<br><br>
2. Об’єкт order
   Створіть об’єкт order:<br>

total — string, наприклад "950"<br>
currency<br>
isPaid — boolean<br>
delivery — "yes" або "no"<br>
priority — "1" або "0"<br><br>
Завдання:<br>
Створіть змінну з числовою сумою замовлення.<br>
Створіть змінну з boolean-версією delivery.<br>
Створіть змінну з boolean-версією priority.<br>
Визначте, чи є замовлення великим (від 1000).<br>
Побудуйте текстовий опис замовлення, враховуючи:<br>
не оплачене → "Order is not paid"<br>
оплачене + доставка → "Paid order with delivery"<br>
велике + оплачене → "High-value paid order"<br>
оплачене + без доставки → "Paid order without delivery"<br>
якщо замовлення пріоритетне — додайте " [PRIORITY]"<br>
(Усю логіку будувати лише через if.)<br>
Зробіть два порівняння суми: total == numberTotal та total === numberTotal, запишіть результати у змінні.<br>
3. Об’єкт systemSettings<br>
   Створіть об’єкт systemSettings:
   <br>
darkMode — boolean<br>
fontSize — string, наприклад "18"<br>
language — "en" або "uk"<br>
betaAccess — string "true" або "false"<br>
Завдання:<br>
Створіть змінну з числовим значенням fontSize.<br>
Створіть змінну з boolean-версією betaAccess.<br>
Створіть змінну isLargeFont (18 і більше).<br>
Побудуйте текст налаштувань (через if):<br>
темна тема + великий шрифт → "Dark mode + large font"<br>
темна тема → "Dark mode"<br>
великий шрифт → "Large font"<br>
інакше → "Default settings"<br>
Якщо користувач має бета-доступ — додайте " (Beta tester)".<br>
4. Інтеграція: фінальний доступ до системи<br>
   На основі ВСІХ трьох об’єктів створіть змінну finalAccess, яка визначає, чи користувач має повний доступ до сервісу.
   <br>
Повний доступ дозволено тільки якщо:<br>

доступ з блоку user дозволений<br>
замовлення оплачене або у користувача достатньо грошей на балансі, щоб покрити суму замовлення<br>
валідні налаштування системи:<br>
шрифт > 12<br>
мова "en" або "uk"<br>
Усі перевірки виконуються лише через if + логічні оператори.
<br>
У консоль потрібно:<br><br>

"Full access granted" або "Access denied"<br>
окремо текст, яка умова заблокувала доступ (user/order/system).<br>
(Використовуйте окремі змінні для проміжних результатів перевірок.)<br>
